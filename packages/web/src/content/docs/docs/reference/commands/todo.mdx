## Purpose

Use `/todo` for structured pause-and-resume work from a checklist file with explicit planning, approval, and queued implementation steps.

## Usage

```text
/todo [@todo-file]
```

## Typical flow

- load `@TODO.md` by default or a provided `@file`
- take the first unchecked item only
- queue `ticket/plan`, show the plan, and require explicit approval before implementation
- queue `dev`, then `commit`, then mark the checklist item complete
- repeat until there are no pending tasks left

## Common tools

- `read`
- `question`
- `session_command(planner, ticket/plan)`
- `session_command(worker, dev)`
- `session_command(worker, commit)`
