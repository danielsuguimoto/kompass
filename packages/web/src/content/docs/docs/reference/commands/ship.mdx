## Purpose

Use `/ship` when you want a fast path from current work to a PR through local orchestration and queued same-session command steps.

## Usage

```text
/ship [base-or-context]
```

## Typical flow

- ensure there is a suitable work branch
- queue `branch`, `commit`, and `pr/create` through `session_command`
- stop on blockers and report `Nothing to ship` when PR creation finds no shippable work

## Common tools

- `session_command(worker, branch)`
- `session_command(worker, commit)`
- `session_command(worker, pr/create)`
