## Purpose

Use `/ship` when you want a fast path from current work to a PR through local orchestration and delegated command steps.

## Usage

```text
/ship [base-or-context]
```

## Typical flow

- ensure there is a suitable work branch
- expand and delegate `branch`, `commit`, and `pr/create`
- stop on blockers and report `Nothing to ship` when PR creation finds no shippable work

## Common tools

- `command_expansion(worker, branch)`
- `command_expansion(worker, commit)`
- `command_expansion(worker, pr/create)`
