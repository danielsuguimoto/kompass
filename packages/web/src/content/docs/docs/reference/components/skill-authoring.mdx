## Behavior

- loads only the related code, skills, docs, evals, and conversation context needed to understand the skill well
- designs the skill as one coherent reusable job with a clear trigger description and sensible defaults
- writes or updates `SKILL.md` concisely, adding support files only when they materially help
- validates naming, frontmatter, file references, and relevant scripts or eval helpers
